
### Java как технология 
**Java** - появилась как альтернатива **C++**. 
Отличие от **C++**:

Упрощенный синтаксис
Интерпритируемость  (**Java** работает на виртуальной машине **JVM**)
Автоматическое управление памятью(Сборщик мусора). 

То есть, суть **Java** была в упрощении написания кода и 
кроссплатформенности (**Java** код работает там где работает **JVM**)

Из-за простоты стал популярен и применялся в множестве сфер:

Веб-разработка(Изначально **Java** должна была быть частью браузеров, отсюда пришло название **JavaScipts**, но не стал) 
Компьютерные игры (Многие игры 2000 -ных, были написаны на **Java**, благодаря ООП)
Приложения для смартфоны на **Android** - **Java** основа **Android**

**Java** - является одним из первых ООП языков.
Всё в языке это классы.

*Инкапсуляция* - ограничение доступа к строке или методы в классе через `private`.
*Наследование* - классы могут наследоваться.
*Полиморфизм* - способность одного интерфейса использоваться для разных типов объектов..

Класс шаблон - Объект экземпляр созданный по его образу
### Геттеры - сеттеры

*Геттер* - получение параметра или метода закрытого `private`

*Сеттер* - изменения поля закрытого `private`

```Java
class Person {
    private String name;
    public String getName() { // Геттер
        return name;
    }
    public void setName(String name) { // Сеттер
        if (name != null && !name.isEmpty()) {
            this.name = name;
        }
    }
}
```

### Android

#### Activity
**Activity** - основной компонент интерфейса. Каждый новый экран - это новая **Activity**

**Жизненный цикл**:

**onCreate()** - появление **Activity**. Открытие приложения.

**onStart()** - **Acivity**, видна, но с ней не получится взаимодействовать.

**onResume()** -  **Activity**, готова к взаимодействию.

**onPause()** - **Activity**, приостановлена

**onStop()** - **Activity**, не видна

**onDestroy** - **Activity**, уничтожается

#### AndroidManifest
**AndroidManifest** - файл в расширением `.xml`, в котором расписано, что будет в приложении.

Инициализируются  **Activity**
разрешения для использования "Железа" устройства (Камера, микрофон, доступ к файлам.) и конфиг приложения.

#### Intent 
**Intent** - объект взаимодействия между компонентами. 
Определяет активность - **Activity**
Взаимодействие данных между компонентами (Передача данных от одного **Activity** другому)

#### View/ ViewGroup

**View** - Это класс из стандартной библиотеки **Android**. Служащий, как основа любого видимого элемента. Для отображения элемента наследуют этот класс

**ViewGroup** - Это невидимый контейнер, внутри которого определяется как будут расположены **View**. Является основой всех контейнеров.
То есть, все **Layout** - это **ViewGroup**. Они от него наследуются и им являются.

#### Адаптеры и списки в Android
**Adapter** - это промежуточный элемент, между, условной DB и
**View**-элементом. Он получает данные из структур, объектов или данных из DB. После чего, создаёт **XML**-макет для элементов и наполняет из соответствующими данными.

**Виды Адаптеров**:

**BaseAdapter** - базовый класс, от него наследуются остальные.
**CursorAdapter** - используется для работы с данными из **DB**
**ArrayAdapter`<T>`** - для работы со списками. 
**SipleAdapter** - для работы со сложным отображением. Две строки в одной.  

**List** - В **Android** списки представлены **View**-элементами. 

**ListView** - Отображает полученные от **Adapter** данные в виде вертикального списка.

**RecyclerView** - Это тоже **ListView**, но с улучшенной адаптивностью и расширяемостью.
